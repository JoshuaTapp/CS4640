Caveats with Optimizers

  • Local Minima
    • With non-convex functions (NNs) we often have many local minima and get stuck in them if our learning rate is too small.
  • Plateaus
    • When we real a plateau during optimization, our gradient becomes very small and our progress will stall.
  • Saddle Points: Most Common because of High-D non-convex functions
    • We find a local minima along one axis and reach a gradient of zero, but the global minima is along another axis.

Adaptive Learning Rate: what if convergence is too slow or sub-optimal?

  1. AdaGrad:
    1. Individually adapts the learning rates of all model parameters by scaling them inversely proportional to the square root of the sum of all previous squared values of the gradient.
      1. where ⊗\otimes ⊗﻿ is squared valued of the gradient
      1. where ⊘\oslash⊘﻿ is element wise division
    1. AdaGrad has a problem of scaling down the learning rate too fast before arriving at a non convex region.
  1. RMSProp:
    1. RMSProp solves the issue with AdaGrad by adding an exponentially decaying average to discard extreme past history.
    1. Accumulates only the most recent gradient by using exponential decay β\betaβ﻿
  1. Adam (Adaptive Moment Estimation):
    1. Combines Momentum and RMSProp scaling
      1. (Momentum) Keeps track of an exponentially decaying moving average of past gradients
      1. (RMSProp) Keeps track of an exponentially decaying moving average of past squared gradients
      1. Where both terms are determined by the gradients

Convolution Neural Nets

Convolution Operation: an operation on 2 multidimensional arrays of REAL values:

  1. Input is an array of data
  1. Kernel is an array of model parameters
  1. The process:
    1. The Kernel scans the Input from left-right and then up-down (one pixel at a time)
    1. During this process, you compute the dot product corresponding to the overlap.
    1. The output is a smaller 2D image that is a representation of the original Input.

Why it Works:

Sparse Interactions

  1. Traditional networks have very dense interaction between layers
  1. CNNs use Sparse interactions between layers.
    1. Kernel is much smaller than the input image (tens vs thousands of pixels)
    1. This leads to fewer parameters, less memory, and improved statistical efficiency
      1. O(m×n)→O(k×n)O(\bold{m} \times \bold{n}) \rightarrow O(\bold{k} \times \bold{n}) O(m×n)→O(k×n)﻿ where k is orders of magnitude smaller

    1. For example, if function g(I(x,y))=I(x−1,y) g(I(x,y))=I(x-1,y) g(I(x,y))=I(x−1,y) ﻿meaning it shifts every pixel of I one unit to the left, then convo(g(I))=g(convo(I))convo(g(I))=g(convo(I))convo(g(I))=g(convo(I))﻿.

High Level Feature Extraction

  1. Stacking Feature Maps
    1. You can stack multiple feature maps by applying multiple kernels to its input to produce a stack of feature maps → capable of detecting multiple features anywhere on in its inputs

      1. repeat
      1. n-th layer learns High-Level Features that are created from the Mid-Level Features

CNN Hyperparameters

Same/Zero Padding

  1. Conv. Op shrinks the Input image
  1. pad the input image with zeros to keep the output image the same size as the input image
  1. NOT the same size as the Padded Image, same size as the ORIGINAL IMAGE.

Stride

  1. Shrink the image faster by striding the image by skipping some of the Input's locations.
  1. Equivalent to how you downsample a output into a smaller new output

Pooling Layer

  1. Subsample the input image to reduce resource use / number of parameters
  1. Each node in a pooling layer is connected to the outputs of limited number of nodes from the previous layer
  1. Pooling node has no weight parameter.
    1. It simply aggregate the inputs by using MAX or AVERAGE function
    1. A 2x2 Block pooling node would reduce the input by 75%
  1. Max Pooling introduces INVARIANCE
    1. Because max pooling units are only sensitive to the max value in the inputs neighborhood, the introduce invariance to small translations (small strides)

Sequential Data

Problems with Modeling Sequential Data

  1. Long-Term Dependencies: We need information from the distant past to make a good prediction on the next word.
  1. Word Order in the Sequence: Bag of words representation does not maintain order of words in a sequence.
  1. Parameter Sharing: Each layer would have separate parameters for the same fixed window of words. This means we will learn different parameters for the same phrase as it appears in different parts of the sequence.

AutoEncoders

  • Why do we care about Z? Because it represents dense representation (knowledge) of the data. Z is therefore a form of compression.
  • We learn the Latent Space by training the model to use features to reconstruct the original data, therefore giving us a unsupervised loss function on the train dataset.

Autoencoders are used for representation learning and are made of the following:

  1. Bottleneck hidden layer: forces network to learn a compressed latent representation
  1. Reconstruction loss: forces the latent representation to encode as much information about the data as possible
  1. Loss function doesn't need labels because it compares the generated x^\bold{\hat{x}}x^﻿ and compares it to the training instance.
  1. Autoencoding = Automatically encoding data

Autoencoder Structure

  • Stacked and hourglass symmetry with respect to the bottleneck layer

Unsupervised Pre-Training

  • Train a stacked auto encoder on unlabeled data
  • When you have decent model, copy the lower level, freeze them, and train the new network with your much smaller labeled dataset.

Generative Adversarial Networks (GANs)

  • We throw away the DISCRIMINATOR when our Discriminative Distribution is UNIFORM!
    • That is, We can no longer tell REAL/FAKE inputs.

Difficulties of training GANs

  • Zero Sum Game - G and D can't both win the game
  • Nash equilibrium: Neither G nor D can improve by changing their strategy assuming the other does not change theirs.
    • Nothing guarantees you can reach Nash Equilibrium
  • Mode Collapse: When generator's output becomes less diverse, they specialize into one type of generation.
  • Hyperparameters: Because of the competition, parameters can oscillate and become unstable (no improvement, cycles). → A lot of effort is spent fine tuning.
  • Experience Replay: Use replay buffer (past images by generator) to train discriminator and not current outputs → this helps prevent hyperparameter oscillation.

Making some guidelines for building a stable convolutional GANs:

  • Use Batch Normalization in both George and David

Deep Q-Networks (DQN)

  • Approximate Q-function and use it to infer the optimal policy, π(s)\pi (s)π(s)﻿
  • Loss function is Expected Square Loss
    • Passes the next state-action into network to calculate Max Q-value
    • Then passes it back to the previous time step to calculate the Loss.
  • Experience Replay Tricks:
    • Store experience (actions, state transitions, and rewards) and create mini-batches from them for the training process
    • Create replay memory table (like replay buffer) of transitions as game experience are played
    • Update Q-Network on random mini-batch of transitions from the replay memory instead from consecutive samples (current state of Q-Network)

Policy Learning (Policy Gradients): learn the stochastic policy function that maps state to action, act by sampling that policy

  • On-policy Policy Gradient (10 M time steps)

Actor-Critic

  • 2 Neural Nets
    • Actor: Policy base network that controls which action to take
    • Critic: Value based DQN that measures how good the action taken is.
  • Critic only has to learn the state-action pairs generated by the Actor Policy
  • Can still use Q-Learning Tricks
  • ADVANTAGE FUNCTION: quantify how much an action was better than expected reward by the critic
    • Actors Reward + Diminished Expected Reward from the Critics model - What the critic thought the reward should have been for this state.

Advantage Actor Critic Algorithm

  • Either 2 network or shared network design
