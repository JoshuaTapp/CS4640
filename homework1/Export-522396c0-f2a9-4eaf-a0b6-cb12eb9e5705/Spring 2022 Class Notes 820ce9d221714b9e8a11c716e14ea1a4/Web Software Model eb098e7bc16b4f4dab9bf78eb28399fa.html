  • Usability
  • Security

Object-Oriented and Event Driven

  • Web Browser and Objects
    • Window Object (tabs)
    • Document Object
  • BOM: Browser Object Model
    • Collection of objects that the browser makes available to us for use with javascript
  • DOM: Document Object Model
    • representation of a document (HTML) with a logical tree structure
    • Allows scripting languages (JS) to access and modify the current page
    • Each node of the tree can have event handlers attached
    • Properties:
      • URL
      • lastModified
      • Title
    • Events
      • Load
      • Click
      • Keypress
    • Method:
      • Write (add things to the model)
      • getElementByID (access an element)
  • The Browser’s View
    • Receives the HTML page and constructs a logical tree
    • When you modify the page, you are modifying the DOM not the source HTML
  • The Server Model
    • Separation of Concerns
      • Presentation (view)
      • Logic
      • Data
    • Data: Assumptions
      • Data Structures change infrequently
        • Types, Org, relation of elements
      • Data Presentation changes occasionally
        • how the data are presented
      • Data Values change frequently
        • contents in memory
    • Server Architectural Styles
      • Multi-tiered Arch
        • Each layer is independent and only communicates with adjacent layers
      • Page-centric Architecture
        • Request are made to a main component and the main component response to clients
        • All the layers interact in a chaining method
        • With a long chain, the delay for response increase
      • Dispatcher Architecture
        • Requests are sent to a dispatcher that then forwards the requests to another component that is needed to build the request
        • Forward vs. Redirect control
          • Forward: The client does not know the dispatcher is using other components
            • Example: reload on purchase causes buy twice
            • POST/REQUEST/GET
          • Redirect: The dispatcher redirects the client to another component to fulfill the request
      • Model-View-Controller (MVC)
        • Separate concerns and divide responsibilities of objects:
          • view: UI (HTML, CSS, JS, UI-building PHP)
          • model: DS, DB (PHP)
          • controller: Business Logic (PHP, JS)
        • Abstraction used in web apps
        • Decreases coupling between layers and components which makes it easier to maintain
        • Divide the implementation work
    • Design Patterns
      • Server-side scripting
        • Using HTML templates and embedded scripts to handle logic and presentation
        • Generate HTML on the server
        • embedded server code inside of HTML pages
      • Server-side Framework
        • server software is organized and structured into tiers
        • Execution on server
        • can be single page or multiple pages with any combination of architectures
        • Example: Laravel (PHP), Symphony (PHP), JSP
      • Single-page Application
        • client-side logic handles much of the user experience
        • Logic is associated with a single HTML page, usually written in JS
        • HTML elements dynamically added and removed through DOM manipulation
        • Technologies
          • Ajax: async request
          • DOM manipulation
          • JSON - syntax for describing JS data objects
          • JQuery - Wrapper library built on HTML standard
        • Example: Gmail
      • Front-end Framework
        • client and server software organized and structured, separate logic from presentation
          • capture model of web app data
          • components separate logic and presentation
          • components dynamically generate corresponding code based on component state
        • Example: Angular, React
