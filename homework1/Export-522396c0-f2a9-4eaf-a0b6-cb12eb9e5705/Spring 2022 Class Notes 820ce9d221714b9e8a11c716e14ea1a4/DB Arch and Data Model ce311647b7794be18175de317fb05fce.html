Data Independence

  • Data independence – upper levels are unaffected by changes to lower levels
  • Logical data independence – changes to logical model do not impact external models
  • Physical data independence – changes to internal model or physical do not impact logical model

Characteristics of Relational Model

  • Originally defined with Set semantics (no duplicate tuples)
  • Attributes are typed and static (INTEGER, FLOAT, ...)
  • Tables are flat (NO subtables or Tables as part of a tuple)
  • Attribute values are atomic
  • Order of tuples doesn’t matter

Structure of Relational Databases

  • Relational Database: a collection of tables, each with a unique name
  • Table: A relation between different attributes encoded by having each row (tuple) represent a relation between the columns (attributes)
  • Tuple: a sequence of values
  • Relation Instance: a specific set of rows
  • Domain: the set of permitted values of an attribute (column)
    • Atomic: a domain is atomic if the elements are considered indivisible units (cannot be split up into sub-elements
  • Null Value: special value that signifies the value DNE or UNKNOWN
  • Database Schema: Logical design of a database
    • Similar to Type Definitions or Class structure
    • EX: department (dept name, building, budget)
  • Database Instance: snapshot of the data in the database at a instance in time.
    • Similar to Class instance

Keys

  • Superkey: set of 1+ attributes that collectively uniquely identify a tuple in a relation
    • May contain extra attributes that are not needed to identify tuple.
  • Candidate Key: A superkey where no proper subset of itself is also a superkey.
  • Primary Key: Candidate key that is chosen as the primary identifying key of the tuples.
    • tuples are prohibited from having the same values for the attributes that make up the primary key
  • Foreign Key: The primary-key used to reference from one relation’s tuple to another relations tuple
    • foreign-key constraint: a primary key for each tuple in one relation must also be the primary key for some tuple in another relation
    • referential-integrity contsraint: relaxes the requirements so that only referencing attributes form the primary key of the referenced relation

Relational Query Languages

  • Query Language: A language that allows a user to request information from the database
  • Imperative QL: User instructus the system on the specific sequence of operations
  • Functional QL: Computation is expressed as eval of function that operate on data in the database.
    • Side effect free / do not change program state
    • Relational Algebra: The theory behind SQL-like languages
  • Declarative QL: User just describes the desired information and system decides how to obtain the data.
    • Tuple Relational calculus
    • Domain relational calculus

Relational Algebra

Is an algebra that is made of the set of operations that combine 1-2 relations as inputs and outputs a new relation as it’s result.
