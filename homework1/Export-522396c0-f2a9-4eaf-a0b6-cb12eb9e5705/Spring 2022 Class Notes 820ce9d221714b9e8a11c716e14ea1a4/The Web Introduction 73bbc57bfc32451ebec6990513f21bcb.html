Web vs Internet

URI: Uniform Resource Identifier

URI: Specify resource by Location and/or name

  • Port number after host name
    • By default 80 / 443 for http / https respectively

URL: Uniform Resource Location

URN: Uniform Resource Name

DNS: Domain Name System

  • maps domain names to IP addresses
  • the phonebook of the Internet
  • domain names are unique, managed by ICANN.

HTTP: HyperText Transfer Protocol

  • Response : Call Protocol
  • Request Types
    • GET: request resource
    • POST: submit data to server
    • HEAD: GET request without response body (Headers only)
    • OPTIONS: describe communication options for resource
      • AJAX queries may check API options
    • PUT: replace resource with payload of request
    • DELETE: delete the specified resource
  • Response
    // Response Line
HTTP/version# status# status

Blank Line
BODY: HTML or raw file (pdf, image, etc)
<!DOCTYPE HTML>
<html><head>...

// Whats next is up to the browser
// Render HTML, display image, etc
    • Response Codes
    • 1xx - Informational
    • 2xx - Sucess
      • 200 - OK - request has succeeded
    • 3XX - Redirects
      • 301 - Moved Permanently - new URL in the response
      • 304 - Not Modified - hasn’t updated (caching)
    • 4XX - Client errors
      • 401 - Unauthorized
      • 403 - Forbidden
      • 404 - Not Found
    • 5XX - Server errors
      • 500 - Internal Server Error
      • 503 - Service Unavailable
  • Properties of HTTP
    • Request-Response cycle
      • Each is completely independent (stateless)
      • interactions always initiated by client request
      • server responds with results
      • Any state information needs to maintained somehow.
        • Must be done outside of http protocol

General Terminology

  • Web page: data that fits on one browser screen
    • Static: HTML already exist on server
      • URL corresponds to a directory location on the server
      • Server’s HTTP response
        • return files or documents stored in that dir
        • if HTML, then browser renders it directly
      • Simple but no interactivity
    • Dynamic: HTML is generated as needed
      • Different content may be displayed each time the page is viewed
      • Two types:
        • Client-side scripting
          • Server response provides code to run locally to determine the final output of the request
          • Generate HTML on the client through scripts
          • Typically Javascript, as source code
          • Advantages:
            • interactive
            • input validation
            • customization
            • improved usability
          • Disadvantages:
            • Browser compatibility
            • mitigated by frameworks
        • Server-Side scripting
          • server responds to HTTP request by running a program that processes the request and produces the response
          • it generate the html at time of request on the server through scripts
          • early approaches emphasized ease of embedding server code inside of HTML pages
          • Example:
          • Advantages:
            • Server produces HTML, so we dont send code to the user. So you get increased security.
            • slightly improved browser compatibility because server is running code not the client
          • Disadvantage:
            • Less interactive - have to go through response cycle for changes. So you do a page reload every time
  • Web Site: a collection of connected web pages
  • Web App: a program that is deployed on the web
    • UI is in HTML (rendered by browser)
    • User interacts through HTTP’s request & response cycle
